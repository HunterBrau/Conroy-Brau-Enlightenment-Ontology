# Raw Data

This folder holds source exports. Pipeline scripts may read these files, but
they should not mutate them in place.

Current raw inputs:

- `18thcentury_french_writers_table.csv`
- `18thcentury_writers_wikidata_viaf.csv`
- `global_writers_1675_1775_discovery.csv`

Current provenance note:

- The two French-seed raw CSVs were manually exported from the Wikidata Query
  Service by a project collaborator.
- The exact original query texts were not preserved in the repository.
- The global writer discovery CSV is generated by
  `scripts/queries/19_discover_global_writers.py`.
- The repository keeps reconstructed French-seed query templates and a query
  runner for provenance, but the active enrichment route is API-first.

Expected API-generated exports:

- `wikidata_affiliation_enrichment.csv`
- `wikidata_person_place_context.csv`
- `wikidata_place_context_entities.csv`
- `global_writers/wikidata_affiliation_enrichment.csv`
- `global_writers/wikidata_person_place_context.csv`
- `global_writers/wikidata_place_context_entities.csv`

Create the global writer/subclass discovery export with:

```powershell
python scripts/queries/19_discover_global_writers.py
```

The discovery query finds humans born 1675-1775 whose occupation is writer or a
subclass of writer. It runs in birth-year chunks because the broad global query
is expensive in the Wikidata Query Service.

Create the Step 04 enrichment export through the Wikidata entity API:

```powershell
python scripts/queries/17_fetch_wikidata_enrichment_api.py
python scripts/queries/17_fetch_wikidata_enrichment_api.py --cohort-id global_writers
```

The API workflow currently supports affiliation evidence, Wikipedia
language-edition representation, gender, ethnic-group context, occupation
profiles, Wikidata label coverage, and sparse notable-work genre/form evidence.
It writes a single CSV that `scripts/pipeline/04_merge_wikidata_enrichment.py`
can consume with no extra arguments.

Local API caches live in `data/raw/cache/`. They are ignored by git because
they are resumable implementation details, not source exports to share.

The person-place context files are generated through the Wikidata entity API:

```powershell
python scripts/queries/18_fetch_wikidata_person_place_context.py
python scripts/queries/18_fetch_wikidata_person_place_context.py --cohort-id global_writers
```

They recover place IDs for birth, death, residence, and work-location evidence,
then attach country and administrative context for the processed
place-affiliation layer.

Reconstructed current-cohort Wikidata queries:

- `scripts/queries/03_wikidata_current_cohort_geography_export.rq`
- `scripts/queries/04_wikidata_current_cohort_viaf_export.rq`
- `scripts/queries/00_run_wikidata_sparql_query.py`

These reconstruction queries are intended to re-export the French seed from
stored QIDs. They improve forward reproducibility, but they should not be
described as the exact original discovery queries unless those originals are
later recovered.
