The flat files in this folder remain the French-seed outputs for backward
compatibility. New France, Germany, British, and China/Qing comparison groups
should use the global context-slice outputs.

These correction tables are generated by:

```powershell
python scripts/queries/12_build_wikidata_label_corrections.py
```

They resolve unreadable Wikidata QID labels found in
`data/interim/writers_cleaned.csv`. They do not modify the cleaned cohort in
place, and they do not resolve duplicate entities or conflicting birth data.

The representation and cultural-affiliation files are generated by:

```powershell
python scripts/analysis/01_build_representation_matrices.py
```

They consume `data/interim/writers_wikidata_enriched.csv`, collapse the cohort
to one row per Wikidata entity, and create the first analysis layer for
Wikipedia language-edition representation. The cultural-affiliation assignment
is provisional: it scores mapped citizenship and language evidence, reports
auditable support counts, and leaves unmapped or context-only tokens in an audit
table.

The place-affiliation files are generated by:

```powershell
python scripts/analysis/02_build_place_affiliation_context.py
```

They consume `data/raw/wikidata_person_place_context.csv`, map birth, death,
residence, and work-location places to provisional country or region
affiliations, and keep place-derived evidence separate from the core
citizenship/language affiliation score.

The geographic-scope files are generated by:

```powershell
python scripts/analysis/03_build_geographic_scope_analysis.py
python scripts/analysis/03_build_geographic_scope_analysis.py --cohort-id global_writers
```

They consume the enriched cohort plus `place_context_long.csv` to count missing
country-of-citizenship evidence, recover China/Qing and British imperial
context through place rows, and classify birth-place context into European,
non-European or colonial, mixed, and transcontinental/imperial buckets.

The cohort-comparison files are generated by:

```powershell
python scripts/analysis/04_compare_cohorts.py
```

They compare the legacy French seed and global writer cohort for high-level
counts, country-of-citizenship tokens, geographic scope, and Wikipedia
language-edition representation. These files are retained for backward
compatibility; new national or imperial comparisons should use the
context-slice outputs.

The political-entity crosswalk review files are generated by:

```powershell
python scripts/analysis/00_build_crosswalk_review_matrix.py
```

They keep manual review labels separate from calculated evidence and add
punchcard-style columns so collaborators can inspect why a British, Qing/China,
France, or Germany entity is included in a given review bucket.

The formula-backed affiliation evidence files are generated by:

```powershell
python scripts/analysis/05_build_affiliation_evidence_matrix.py
python scripts/analysis/05_build_affiliation_evidence_matrix.py --cohort-id global_writers
```

They tally eight explicit evidence fields for each person and candidate
affiliation. The main score is `supporting_evidence_count /
total_evidence_fields`; the available-data score is
`supporting_evidence_count / available_mapped_evidence_count`.

The occupation-bucket files are generated by:

```powershell
python scripts/analysis/06_build_occupation_bucket_tables.py
```

They map occupation QIDs to granular review buckets, including a dedicated
Religion / Theology bucket, then summarize bucket coverage and Wikipedia
language-edition representation for each cohort.

The context-slice files are generated by:

```powershell
python scripts/analysis/07_build_context_slice_tables.py
```

They derive France, Germany, British, and China/Qing slices from the global
cohort and the political-entity crosswalk. The accompanying
`french_seed_redundancy_audit.csv` confirms that the legacy French seed is
fully contained in the global cohort and should be treated as provenance rather
than active discovery data.

The core findings packet files are generated by:

```powershell
python scripts/analysis/08_build_core_findings_packet.py
```

They summarize the current scoped project in reusable tables and write the
human-readable packet to `docs/core_findings_packet.md`.
